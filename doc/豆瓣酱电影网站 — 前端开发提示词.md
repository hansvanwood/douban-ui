# 豆瓣酱电影网站 — 前端开发提示词

------

## 项目概述

开发一个名为**豆瓣酱**的电影展示网站，使用 **Vue 3 (Composition API) + Vue Router 4 + Pinia + Axios** 技术栈，后端 Base URL 为 `http://localhost:8080/api`。

页面风格参考 **Emby 媒体服务器**：深色背景、卡片式布局、强调色为绿色 `#60A84E`，整体质感现代、沉浸。

------

## API 封装

### request.js（Axios 实例）

- `baseURL: 'http://localhost:8080/api'`
- 响应拦截器：若 `response.data.code === 500`，自动跳转 `/500`；若 `code === 404`，抛出 404 错误供页面处理
- 请求拦截器：统一设置 `Content-Type: application/json`

### movie.js

```js
export const fetchMovieList = (params) => request.post('/movies/list', params)
export const fetchMovieStats = () => request.get('/movies/stats')
export const fetchMovieDetail = (movieId) => request.get(`/movies/${movieId}`)
export const fetchMovieRegions = () => request.get('/movies/regions')
export const fetchMovieLanguages = () => request.get('/movies/languages')
export const fetchMovieTypes = () => request.get('/movies/types')
```

### comment.js

```js
export const fetchComments = (movieId, pageNum = 1, pageSize = 20) =>
  request.get(`/comments/${movieId}`, { params: { pageNum, pageSize } })
```

### worker.js

```js
export const fetchWorkerDetail = (workerId) => request.get(`/workers/${workerId}`)
```

## 公共组件规范

### AppHeader.vue（顶部导航栏）

- 固定在页面顶部（`position: fixed; top: 0; z-index: 100`）
- 左侧：Logo 文字「🎬 豆瓣酱」（点击跳转首页），使用 `--color-primary` 配色
- 右侧导航链接：「首页」「电影」，当前激活项用 `--color-primary` 下划线高亮
- 背景：深色半透明 + `backdrop-filter: blur(10px)`

### Breadcrumb.vue

- 接收 `items: Array<{label, to}>` prop
- 用 `>` 分隔，最后一项不可点击
- 在 AppHeader 正下方，`position: sticky; top: 60px` 跟随页面滚动

### MovieCard.vue

- 接收 prop：`movie: Object`
- 海报图：宽高比 2:3，封面图路径使用后端返回的cover拼接，规则为${baseURL}/poster/${cover}
  - 若 `cover` 为空，显示占位灰色卡片 + 🎬 图标
  - 图片加载失败时同样降级显示占位
- 右上角：豆瓣评分（金色 `--color-gold`，字体加粗）+ ⭐
- 下方信息区：片名（大字，单行省略）、主演（最多3人，灰色小字）、类型标签（绿色胶囊）、年份
- hover 时卡片上移 `translateY(-4px)` + 阴影加深，海报图轻微放大 `scale(1.03)`，过渡 `0.2s`
- 整个卡片可点击，使用 `router-link` 或 `$router.push`

### StarRating.vue

- 接收 `score: Number`（0-10，豆瓣评分），`size: String`（'sm'|'md'|'lg'）
- 豆瓣评分 ÷ 2 = 5星满分；渲染5个星形，支持半星显示
- 金色实心星 / 半星 / 灰色空星

### Pagination.vue

- 接收：`currentPage, totalPages, total`
- 最多显示7个页码，超出用 `...` 省略
- 「上一页」「下一页」按钮，边界时禁用
- 支持输入框跳转指定页（输入回车确认）
- 显示「第 X 页 / 共 Y 页，共 Z 条」

### CountUp.vue

- 接收 `target: Number`（目标数字）
- 组件挂载后用 `requestAnimationFrame` 做 1.5s 数字递增动画，从 0 计数到 `target`

### EmptyState.vue

- 接收 `message: String`
- 居中显示大号空状态图标（可用 SVG 内联）+ 提示文字

### 2. MovieListView.vue — 电影列表页

**布局**：左右双栏（左侧固定筛选栏 260px + 右侧主区域自适应）

筛选项与接口对应关系：

| 筛选项       | 接口字段                  | 数据来源              |
| ------------ | ------------------------- | --------------------- |
| 关键词搜索框 | `keyword`                 | 用户输入，防抖 500ms  |
| 类型下拉     | `type`                    | GET /movies/types    |
| 语言下拉     | `language`                | GET /movies/languages |
| 地区下拉     | `region`                  | GET /movies/regions   |
| 年份输入框   | `year`                    | 用户输入              |
| 最短时长     | `minMinutes`              | 用户输入              |
| 最长时长     | `maxMinutes`              | 用户输入              |
| 排序选择     | `sortField` + `sortOrder` | 用户选择              |

**排序选项映射**：

```
「豆瓣评分从高到低（默认）」→ sortField: 'douban_score', sortOrder: 'desc'
「豆瓣评分从低到高」       → sortField: 'douban_score', sortOrder: 'asc'
「上映时间从新到旧」       → sortField: 'release_date', sortOrder: 'desc'
「上映时间从旧到新」       → sortField: 'release_date', sortOrder: 'asc'
「电影时长从长到短」       → sortField: 'minutes',      sortOrder: 'desc'
「电影时长从短到长」       → sortField: 'minutes',      sortOrder: 'asc'
「片名 A-Z」              → sortField: 'movie_name',   sortOrder: 'asc'
```

「搜索」按钮：重置 pageNum=1，触发请求；「重置」按钮：清空所有筛选项，触发请求

**右侧电影列表区**：

- 标题行：「共找到 XXX 部电影」（灰色小字）
- 网格布局：`display: grid; grid-template-columns: repeat(auto-fill, minmax(180px, 1fr)); gap: 20px`

- 无结果时显示 `EmptyState` 组件
- loading 状态：显示骨架屏（灰色占位卡片动画）

**底部分页器**：使用 `Pagination` 组件，居中显示

**URL 同步**：筛选条件同步到 URL query 参数，支持分享链接和浏览器前进/后退，页面初始化时从 URL 读取参数初始化筛选状态

## 重要细节与注意事项

**图片处理**：

- 电影列表接口返回的 `cover` 字段已经是完整 URL（如 `https://img3.doubanio.com/...`），直接作为 `<img>` 的 `src` 使用，**不需要**拼接 baseURL
- 图片加载失败时（`@error`）降级为占位状态

**字段解析函数（建议提取到 utils/parse.js）**：

```js
// 解析 actorIds / directorIds: "姓名:id|姓名:id" → [{name, id}]
export function parseWorkerIds(str) {
  if (!str) return []
  return str.split('|').map(item => {
    const [name, id] = item.split(':')
    return { name, id: Number(id) }
  })
}

// 哈希取颜色
const COLORS = ['#60A84E','#4a90d9','#e67e22','#9b59b6','#e74c3c','#1abc9c','#f39c12','#2980b9']
export function hashColor(id) {
  return COLORS[Number(id) % COLORS.length]
}

// 格式化日期（去掉时分秒）
export function formatDate(dateStr) {
  if (!dateStr) return ''
  return dateStr.slice(0, 10)
}
```

**Pinia Store 建议**：

- `movieStore`：存储当前电影列表数据、筛选参数、分页状态，避免路由切换后重复请求
- `filterStore`：持久化筛选条件（types/regions/languages 枚举数据，只请求一次）

**加载状态**：每个页面有全局 loading 状态，显示骨架屏或 spinner，避免白屏

**错误处理**：

- 404（电影/工作者不存在）：页面内显示友好提示，不跳转
- 500（服务器错误）：Axios 拦截器自动跳转 `/500`
- 网络错误：显示 toast 提示「网络连接失败，请检查后端服务」

**性能优化**：

- `MovieCard` 图片使用 `loading="lazy"`
- 路由级别懒加载所有 View 组件：`component: () => import('./views/HomeView.vue')`
- 枚举数据（types/regions/languages）在应用初始化时一次性获取，存入 Pinia

## 验收标准

1. 所有页面在 1280px 宽度下布局正常，移动端（375px）可正常使用
2. 筛选 + 分页 URL 参数同步，刷新页面状态保持
3. 点击演员/导演姓名正确跳转到对应工作者详情页
4. 图片加载失败有降级占位，无结果有空状态组件
5. 评论分页切换后滚动到评论区顶部
6. 首页数字有 CountUp 动画效果
7. 路由切换顶部导航高亮同步
